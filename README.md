### I. ЧТО ЭТО

Полный машинный корпус священных текстов ветвей, разобранных в `archivum-haereticum`: **весь текст целиком** (не выдержки) в оригинале — с параллельным русским переводом, с указанием издания и лицензии каждого слоя. Собирается автоматически из открытых достоверных источников; сайт генерируется и публикуется сам (GitHub Actions → GitHub Pages).

### II. ANTI-CONSPIRATIO — ХАРТИЯ ФАЛЬСИФИЦИРУЕМОСТИ

Этот проект — не теория заговора, и вот его конституционная гарантия:

1. **Каждый слой несёт источник и печать.** Нет источника — нет слоя.
2. **Протокол опровержения публикуется.** Архив уже казнил собственную гипотезу (иероглифная криптография, протокол 0/5 — см. `archivum-haereticum/05-probatio/damnata.md`). Проект, который не может быть опровергнут, — заговор; проект, который публикует способ своего опровержения, — исследование.
3. **Граница двух корпусов проведена явно.** *Исторический корпус* (тексты мигрировали, редактировались, цитировали друг друга) — доказуем и частично доказан: это текстология. *Метафизический корпус* (единый Автор) — нефальсифицируем, это теология; печать SUSPECTUS на нём стоит навсегда. Свод документирует, **как люди собирали книги, приписываемые Богу**; есть ли за ними Автор — Архив печати не имеет.
4. **Закрытый вопрос запрещён.** Закрытый вопрос перестаёт приносить данные.

### IV. КАК ИСПОЛЬЗОВАТЬ НЕЗАВИСИМО

- **Человек:** включить Pages (Settings → Pages → Source: *GitHub Actions*) — после первого прогона сайт живёт сам, обновляется при каждом пуше. Либо `git clone` и открыть `site/index.html` локально.
- **Машина:** читать JSONL из `data/` напрямую (raw.githubusercontent) — схема в `SPEC.md`; каждая строка — один стих с источником и печатью.
- **Интеграция в Heretic OS:** модуль LIBER читает `site/`, модуль NOOSPHERE — `data/`; MCP-сервер Архива (`archivum-haereticum/mcp/`) получит endpoint `corpus` в фазе Q-связка.
